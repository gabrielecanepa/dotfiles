# AGENTS.md

Single source of truth for all coding agents (Claude Code, Codex, Copilot). This file is symlinked from `~/.claude/CLAUDE.md` (Claude Code auto-loads it every session) and read by Codex via the AGENTS.md convention. Copilot reads `~/.github/copilot-instructions.md` + `~/.github/instructions/`. Tone/behavior rules live in `.agents/rules/behavior.instructions.md` (not duplicated here) — read both.

## Environment

- **User:** set with `$NAME`/`$EMAIL` in `.zprofile`.
- **Machine:** macOS, Apple Silicon (arm64). Homebrew prefix is `/opt/homebrew`.
- **`$HOME` IS a git repo** (branch `main`). These dotfiles are version-controlled. Editing any file under `~` may be a tracked change — treat it like a repo edit, not a throwaway.
- **Project workspace:** `$WORKING_DIR` = `~/Developer`. Real project code lives there, not in `~`. `cdw` jumps to it.
- **Editor:** VS Code (`EDITOR=code`, `GIT_EDITOR="code --wait"`). **Terminal:** Ghostty.

## Shell

- **zsh + oh-my-zsh**, but `ZSH_CUSTOM=~/.zsh` with a **custom plugin/theme framework** under `.zsh/plugins/*` and `.zsh/themes/*`. Many shell "commands" are custom functions/plugins, not binaries (`profile`, `plugin`, the `brew` wrapper, `lts`, `completions`, `deps`, etc.). Check `.zsh/plugins/<name>/` before assuming a command is a system tool.
- Startup: `.zshenv` (env, Homebrew + `*ENV_ROOT` vars, PATH via `_init_path`) → `.zprofile` (GENERATED by the `profile` plugin; identity vars only — never hand-edit) → `.zshrc` (oh-my-zsh, plugins, completions) → `.aliases`. Put new env/PATH in `.zshenv`.
- The `brew` command is **wrapped** (`.zsh/plugins/brewfile`): `brew dump` writes the Brewfile, `brew global` installs from it, `brew fresh` updates+upgrades+cleanup+dump+doctor. Brewfile is at `.homebrew/Brewfile`.
- `.zshrc` auto-symlinks `.husky/*` hooks into `.git/hooks/` on shell start when `profile check` passes — do not hand-edit `.git/hooks/`.

## Toolchains (version managers — NOT mise/asdf/nvm)

- **Node:** `nodenv` (pinned version in `.node-version`). **Default package manager: `pnpm`** — use it for all generated install/run/exec commands (`pn`=`pnpm`, `pnx`=`pnpx`). `bun`, `npm`, `corepack`, `deno` are also installed; only use them if a project's lockfile/config calls for them (`bun.lockb`→bun, `package-lock.json`→npm).
- **Python:** `pyenv` (pinned version in `.python-version`).
- **Ruby:** `rbenv` + `ruby-build` (pinned version in `.ruby-version`).
- Assume these shims are on PATH; do not invoke system `python3`/`ruby`/`node` directly.

## Git & commits

- **Commits are signed** via the 1Password SSH agent (`gpg.format=ssh`, `commit.gpgsign=true`, signer `op-ssh-sign`). Don't disable signing or add `--no-gpg-sign`.
- **No `Co-authored-by` / AI trailers** (`git.addAICoAuthor=off`). Keep messages single-line unless a `BREAKING CHANGE:` footer is needed.
- **Conventional commits are enforced** by commitlint via husky `commit-msg`, re-checked on `pre-push` (which also runs `shellcheck` on shell dotfiles and `oxfmt --check`). Every commit message MUST pass `.commitlintrc`:
  - Format: `<type>(<scope>)?: <subject>` — type + subject required, lower-case, no empty subject.
  - **This repo's allowed types (non-standard):** `ai`, `brew`, `chore`, `docs`, `git`, `node`, `python`, `ruby`, `shell`, `vscode`. Only allowed scope: `npm`. Do **NOT** use `feat`/`fix`/`refactor` here.
  - VS Code Copilot/GitLens are separately configured to emit _standard_ `feat:`/`fix:` style — that guidance is for **other** projects, not this dotfiles repo. **Match the repo you're in.**
- `pull.rebase=true`, `push.autoSetupRemote=true`, `init.defaultBranch=main`.
- **Rich git aliases exist — prefer them** (in `.gitconfig`, also exposed as `g<alias>` shell aliases): `gst` (status -sb), `gacm` (add -A && commit -m), `gcm`, `gco`, `glg`, `gsweep` (delete gone branches), `gfps` (fetch all + pull --rebase + sweep), `gpristine`, `gredo`, `gundo`. Run `git aliases` to list all.

## Formatting & linting (respect existing config; don't reformat to your own style)

- **JS/TS/CSS/JSON:** `oxfmt` (`.oxfmtrc.json`) — no semicolons, single quotes (double in CSS), `es5` trailing commas, avoid arrow parens; imports non-relative.
- **Ruby:** `rubocop` (`.rubocop.yml`) — double-quoted strings, line length 120, `NewCops: enable`.
- **Shell:** `shellcheck` (`.shellcheckrc`).
- **All files:** `.editorconfig` — UTF-8, LF, 2-space indent, final newline, trim trailing whitespace, max line 120.

## Conventions / do & don't

- **Single source of truth for agent config:** edit `.agents/{AGENTS.md,rules,skills,hooks}`. `.claude/*`, `.codex/{AGENTS.md,skills}`, and `.github/copilot-instructions.md` resolve to `.agents` — never edit symlink targets directly or duplicate content across agents.
- **`$HOME` uses an allowlist `.gitignore`** (`/*` then `!`-unignores). **A new file under `~` is NOT tracked unless you add a matching `!` rule to `.gitignore` in the same change.**
- **New-machine bootstrap:** `.github/install.sh` shallow-clones the repo over HTTPS into an auto-cleaned temp dir and installs each tracked file into `~`, prompting before it overwrites an existing one. Replaced files are backed up (path-mirrored) to `${XDG_STATE_HOME:-$HOME/.local/state}/dotfiles/backup/<timestamp>/` — restore one by copying it back from there. After the script, `~` is not yet a git repo; initialize it with `git -C ~ init -b main && git -C ~ remote add origin git@github.com:gabrielecanepa/dotfiles.git && git -C ~ fetch --depth 1 origin main && git -C ~ reset --hard FETCH_HEAD` (see README §Git repository).
- **Prefer existing aliases/functions over raw commands** — `.aliases` and the git aliases are curated; use `gst`, `pn`, `cdw`, `brew fresh`, etc.
- Don't introduce `mise`, `asdf`, `nvm`, or `volta` unless explicitly asked — the version-manager stack is nodenv/pyenv/rbenv and is intentional.
